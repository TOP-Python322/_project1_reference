=========  МОДЕЛЬ РАСПРОСТРАНЕНИЯ  ========

Документированные файлы с кодом и файлы данных.
Файл зависимостей requirements.txt



=============  МОДЕЛЬ ДАННЫХ  =============

1. Структуры данных
    
    # база игроков - имена и статистика игроков 
    
    {
      '<имя_игрока1>': { 
        'wins': <количество_побед>,
        'fails': <количество_поражений>,
        'ties': <количество_ничьих>      
      },
      '<имя_игрока2>': {
        'wins': <количество_побед>,
        'fails': <количество_поражений>,
        'ties': <количество_ничьих>      
      },
      ...
    }
    
    # сделанные ходы
    
    [4, 2, 8, 0]
    
    {4: 'X', 2: 'O', 8: 'X', 0: 'O'}
    
    # база сохранений - сохранённые партии
    
    {
      frozenset({'<имя_игрока1>', 
                 '<имя_игрока2>'}): {
        'X': '<имя_игрока1>'
        'turns': [4, 2, 8, 0],
        'dim': 3,
      },
      frozenset({'<имя_игрока3>', 
                 '<имя_игрока1>'}): {
        'X': '<имя_игрока3>'
        'turns': [0, 4, 6, 7, 2],
        'dim': 3,
      },
      ...
    }

2. Файлы данных
    
    # имена и статистика игроков 
    
    [<имя_игрока1>]
    wins = <количество_побед>
    fails = <количество_поражений>
    ties = <количество_ничьих>

    [<имя_игрока2>]
    wins = <количество_побед>
    fails = <количество_поражений>
    ties = <количество_ничьих>

    # сохранённые партии
    
    <имя_игрока1>,<имя_игрока2>!4,2,8,0!3
    <имя_игрока3>,<имя_игрока1>!0,4,6,7,2!3



=====  ПОСЛЕДОВАТЕЛЬНОСТЬ ВЫПОЛНЕНИЯ  =====

1. Чтение файлов данных

2. ЕСЛИ первый запуск:
        вывод раздела помощи 

3. Запрос имени игрока
     а) ЕСЛИ введённое имя отсутствует в базе игроков:
             добавление записи о новом игроке в базу игроков
             обновление файла данных игроков
     
     б) добавление введённого имени в список активных игроков

# суперцикл (главное меню)
    
    4. Ожидание ввода команды игрока
    
    # new
        
        # настройка игрового процесса
        
        5. Запрос режима игры
            
            6. ЕСЛИ выбран режим один человек:
                    
                    7. Запрос уровеня сложности бота
                         ЕСЛИ выбран лёгкий:
                              добавление имени "#1" в список активных игроков
                              запись ссылки на объект функции для лёгкого бота в отдельную переменную
                         ИЛИ ЕСЛИ выбран сложный:
                              добавление имени "#2" в список активных игроков
                              запись ссылки на объект функции для сложного бота в отдельную переменную
            
            8. ЕСЛИ выбран режим два человека:
                    повтор шага 3
        
        9. Запрос очерёдности хода
        
        # игровой процесс
        
        10. Цикл до максимального количества ходов:
            
            11. Запрос хода игрока
                  а) ЕСЛИ ввод пустой:
                          добавление записи о текущем состоянии партии в базу сохранений
                          обновление файла сохранений
                          переход к шагу 4
                  
                  б) ЕСЛИ ввод некорректный:
                          вывод сообщения о некорректном вводе
                          переход к шагу 11
                  
                  в) ЕСЛИ ввод корректный:
                          обновление структуры данных для ходов в текущей партии
            
            ИЛИ
            
            11. Вычисление хода бота и обновление структуры данных для ходов в текущей партии
            
            12. Отображение сделанного хода
            
            13. Проверка на наличие победной комбинации
                  ЕСЛИ победная комбинация найдена:
                       вывод сообщения о победе игрока или бота
                       переход к шагу 15
                  ИНАЧЕ:
                       переход к шагу 11
        
        14. Вывод сообщения о ничьей
        
        15. Обновление базы игроков, обновление файла данных игроков, возвращение списка активных игроков к состоянию до ввода команды new, сброс структуры данных для ходов
        
        16. Переход к шагу 4
    
    # load
        
        # настройка игрового процесса
        
        17. Запрос слота сохранения
        
        18. Обновление списка активных игроков, обновление структуры данных для ходов, обновление глобальных переменных, связанных с размерностью
        
        # игровой процесс
        
        повтор шагов 10–16
    
    # quit
        
        19. Переход к шагу 20

20. Действия перед завершением работы приложения
